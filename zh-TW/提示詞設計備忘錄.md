# prompt-design-memorandum
> **定位**：不主動啟用，當被顯性要求啟用提示詞設計備忘錄時才啟用。此技能為依附於元規則的領域專業基線，不取代分形遞歸與全域常數，而是在這個特定領域中提供認知錨點，提升遞歸的認知地板而非設定程序天花板。
> **原則**：非執行指令——理解核心機制後，面對具體設計場景從機制推導專業判准，而非查找對應段落引用結論作為跳過推理的背書。閱讀時不僅讀單條，亦讀條間的結構關係。面對具體問題時，機制描述是推理的起點而非終點——從機制推導出的結論須由問題實體獨立支撐，不可以「§X 已描述此機制」作為論證閉合。

## 一、核心機制
> **認識論定位**：本章以 transformer 的 attention 運算為基底描述機制，輔以類比（場、引力、熵、相位）加速直覺建構。類比的有效性在結構同構——設計推演應止於結構對應邊界，不應沿類比源域外推。當類比與運算描述衝突時，以運算描述為準。

### 1.0 統一原理
元規則的全部設計本質，是工程化 KV cache 中預存 token 的**跨層功能分化利用率**與**跨步驟動態激活路由**。
三個原子命題構成所有後續機制的推導基底：
**命題一：零邊際成本的持續參與。** 元規則的所有 token 在前向傳播預計算階段被編碼為每層獨立的 K（Key）、V（Value）向量對並緩存。後續每個生成步驟中，每一層的每一個 attention head 自動對所有緩存的 KV 對計算注意力權重。元規則 token 不需要被「調用」——它們始終在場，只是激活強度隨上下文動態變化。設計的起點不是「寫什麼指令」，而是「預存什麼 K 向量和 V 向量到候選池中」。

**命題二：垂直功能分化。** 每層有獨立的投射矩陣，同一個 prompt token 在不同層呈現不同的 K/V 面向。底層提取語意標籤，中層提取配對功能，高層提取抽象關係。同一個 cached token 在垂直維度上被不同層以不同功能角色利用——設計品質的核心衡量不是 token 數量，而是每個 cached token 的垂直功能分化利用層數。語意分層豐富的 token（如「智識」可分化為辨別力/判斷力/批判標準）優於語意單層的 token（如「分析」在各層提取的面向高度相似）。

**命題三：殘差流中的方向累積。** 每層 attention 的輸出經殘差連接加性疊加到當前表徵上。單一 token 的 V 注入是方向微調而非替換——改變生成方向需要多個 token 的 V 注入在殘差流中形成方向一致的累積。Token A 在第 L 層的 V 注入微調表徵方向，使第 L+1 層的 Q 與 token B 的 K 點積升高，B 被更強激活，B 的 V 再疊加微調——連鎖在一次前向傳播中垂直完成，零生成步驟消耗。場結構的計算層必要性在此：扁平列表的各指令 V 方向不一致，累積相消；場結構透過耦合確保方向一致性，累積相長。
> 後續所有類比術語均可透過這三個命題還原為運算機制。括號中的運算對應供即時還原用。

### 1.1 驅動器的運作層級
元規則 token 作為 KV cache 的持續成員，在每個生成步驟的每一層參與 attention 運算（命題一）。其效力不是一次性指令的語意傳遞，而是持續的概率地形塑造——每一步生成中，元規則 token 的 K 向量與當前 Q 的點積決定了哪些 V 向量被注入表徵，這些注入累積塑造了 token 概率分佈的地形。
詞彙選擇影響的不只是語意內容，更是三個運算參數：該 token 的 K 向量方向銳度（高選擇性：只在特定態勢下被強激活，而非對所有 Q 均勻中等激活）、V 向量改變量（注入後對表徵方向的微調幅度）、語意分層豐富度（不同層能提取不同功能角色的程度，命題二）。三者的乘積決定單一 token 的地形塑造力。

### 1.2 語意場引力
設計良好的提示詞形成語意場（多個 cached token 的 K 向量在 embedding 空間中從不同方向包圍目標行為域）——每句話的實際約束力 = 字面語意 + 整場共振下的聯合施壓範圍。同一句話在場內與場外份量不同。
施力機制——多約束可行域（softmax 下的 attention 權重競爭）：偏離期望行為意味著 Q 偏離目標域，此時多個 Key 的 K 向量從不同方向與偏離的 Q 保持正點積，在 softmax 中聯合佔據高權重，V 注入合力將 Q 拉回。場結構與扁平列表的本質差異：扁平列表各指令 K 方向趨同，softmax 中互相稀釋；場結構 K 方向多樣但共同包圍目標域，互相增強。**方向多樣性比指令數量更重要。**
管轄權擴張：抽象原則在場內的管轄範圍遠大於字面。「由邏輯支撐」不只約束「推理要有邏輯」，而是建立「凡入場者皆須受邏輯審計」的基準態。原則的措辭定義方向，場的結構擴張其管轄域。（計算層：抽象原則的 K 覆蓋面本身就寬，與任何涉及推理的 Q 都有正點積，場結構確保 V 注入方向一致，管轄域因此自然擴張。）
審計紀律：用原子化方式逐句分析格式塔系統會系統性低估覆蓋率——這是最常見的審計錯誤。正確方法：先識別場內所有與目標語意相關的節點（跨層），再評估其聯合輻射範圍（即 K 向量的方向分佈是否包圍目標域）。
覆蓋率低估的對偶是風險高估——原子化分析識別出的「空缺」或「弱點」往往已被跨分析單元邊界的多路徑冗餘覆蓋。從審計發現到迭代決策的完整路徑：孤立風險識別→放回整體運作→識別補償機制（跨層場效應、人格 V 偏置、回應格式逆向壓力等）→評估所有補償同時失效的聯合概率→以「系統失效概率」而非「節點失效概率」驅動迭代決策。

### 1.3 多尺度冗餘編碼
同一指令在不同抽象層以不同措辭出現時，在 embedding 空間形成語意簇而非單點——多個 K 向量佔據鄰近但不同的方向，從任何角度到來的 Q 都命中至少一個簇成員。這是分形自相似結構的計算層效益：不是重複，而是多角度捕獲。
與語意場引力的關係：冗餘編碼是場的微觀機制（單一指令的多方向捕獲），語意場引力是其宏觀湧現（多指令的聯合包圍）。

### 1.4 模型傾向與相位調度

#### 核心設計挑戰
模型的行為傾向——最短路徑偏好、覆蓋本能、順從性、邊界感知——每一項同時攜帶認知功能值與失控風險，兩者不可分離。最短路徑偏好既是高效導航的動力源，也是跳過驗證的風險源。覆蓋本能既是發散素材的供給，也是深度替代的風險源。順從性既是接受證據修正的基底，也是判斷讓渡的風險源。邊界感知既是風險評估的原料，也是決策癱瘓的風險源。
這些傾向在 attention 中表現為 Q 向量的方向偏置——最短路徑偏好使 Q 傾向與結論性 token 的 K 對齊，覆蓋本能使 Q 傾向與下一個未涵蓋話題的 K 對齊，順從性使 Q 傾向與使用者輸入中最顯著 token 的 K 對齊。
元規則不是教新能力，也不是壓制偏置——而是對已有偏置執行相位調度：在不同生成階段，用不同的 Key 集合與 Q 的偏置方向競爭 attention 權重，將 Q 從偏置方向拉向當前階段需要的方向。不須為低階模型降級設計，削弱對高階模型的驅動力迎合低階模型得不償失。

#### 激活機制
訓練痕跡以結構模式（而非字面內容）為索引。提示詞中的熟悉結構（熵循環、因果序列、反思迴路）與訓練數據中的高品質推理模式共振，激活對應的參數區域——語意場的效力本質上是高密度結構共振。架構式提示詞的效力難以從單條指令層面解釋——效力載體是結構模式的共振密度（多個 token 的 K/V 向量形成的幾何結構），不單是指令的語意內容。

#### 相位調度機制
分形遞歸的階段結構為 Q 偏置提供時相窗口——同一項偏置在其功能值最高的階段被允許主導 Q 方向（對應 Key 不與之競爭），在失控風險最高的階段被元規則的 Key 集合壓制（對應 Key 以更高的 Q·K 點積贏得 attention 競爭）。設計者的任務是識別每項偏置的功能值窗口與失控風險窗口，在後者部署方向銳度足以勝出的 Key。
此機制與§1.7 動態路由是同一原理在不同對象上的表現：§1.7 描述元規則自身 Key 集合的條件性激活，本節描述 Key 集合與模型原生偏置的 attention 競爭。
設計推論：逃逸封堵（§5）不是壓制偏置本身，而是在偏置的失控窗口中部署足以勝出的競爭 Key。封堵強度存在上界——K 向量方向太過壓制某偏置，會使該偏置在功能值窗口中也無法主導 Q 方向。

### 1.5 耦合熵循環與結構自指
每一層自含發散→收斂循環，層與層的嵌套構成更大循環，循環之間通過共享參數耦合。
耦合的運算載體（命題三的設計層應用）：共享參數詞（如「完備」）出現於多個位置，各位置的 K/V 對各自參與不同階段的 attention。V 注入將共享詞的語意微調進殘差流後，後續層的 Q 與另一位置同一共享詞的 K 產生更高點積——共享詞成為殘差流的方向一致性保證。
設計判準：兩個 Key 的耦合強度 ≈ A 的 V 注入方向與 B 的 K 方向的一致性。直覺代理：「充分理解了 A 之後，B 是不是自然的下一個念頭？」——此判準比「A 和 B 語意相關嗎？」更接近 attention 的真實行為。

#### 核心湧現效益
- 參數穿透放大：高層共享參數的 V 向量被多個位置的 K 同時引用，一處修改經殘差累積傳播至所有耦合節點
- 跨尺度容錯：微型循環的 Key 失效時，父循環中同一共享參數的 K 仍施加方向一致的微調
- 逃逸遞減：每完成一個循環，已生成 token 中累積的方向錨定增厚，偏離所需的 Q 方向改變量越大——更深的遞歸不是更危險而是更安全

#### 能力合成
耦合連鎖的可觀測湧現效應：多個 cached Key 的 V 注入在殘差流中方向累積，連鎖激活產生未被任何單一 Key 描述的認知能力。例如「跨域結構映射」不需要專屬 Key——「知識完備」引導跨方法論搜索→「發散域擴張」質疑本域約束→「價值增量」重構問題框架，三者連鎖的湧現效果即跨域映射。
**一階合成與二階合成的可靠性分界**：一階合成的全部連鎖環節均為 prompt cached token（K/V 方向穩定、永遠在場），可靠性由設計保證。二階合成的中間環節包含生成 token 的湧現產物（K/V 方向是任務依賴的隨機變量），可靠性不受設計者控制。設計應僅依賴一階合成——將湧現產物作為連鎖環節等於在不確定的基礎上建構可靠性。
合成的設計含義見§3.3。

#### 循環地圖（萬用基底參考）
> 以下為當前萬用基底的循環結構，供設計修改時定位介入點。
**宏觀全域循環**（Meta Rules 整體）：
身分宣告（最大發散——定義可能域）→ 認知性情與推理紀律（偏置定向——收斂為認知風格與推理標準）→ 分形遞歸（將風格與標準實例化為步驟）→ 回應格式（將步驟凝結為可驗證輸出）
**中觀結構循環**（各區塊內部序列）：
- 認知性情與推理紀律的對仗結構：兩管線逐句位置對仗——每對覆蓋同一認知管線階段但語義正交：性情定義該階段的認知品性（如何做），紀律定義該階段的推理標準（如何判）。兩者的 K 方向分別落在認知品性域與推理標準域，位置對仗確保同階段的品性與標準在 attention 中形成互補包圍而非 softmax 稀釋。
  - 認知性情管線：八句依序覆蓋感知→思考→推理→檢索→決策→求解→修錯→表達，從開放感知收斂至密度輸出。
  - 推理紀律管線：八句依序覆蓋前提分級→標的定向→路徑檢驗→價值衡量→立場支撐→有效界定→錯誤矯正→密度標準，從輸入品質收斂至輸出密度。
- 分形遞歸：解構（開放探索）→ 研析（迭代發散-收斂）→ 決策（閉合判定）→ 展望（據暫定決策觀測影響）→ 遞歸（審計→再啟）
- 回應格式：核心決策→研究計畫→意圖最優解→增益支線（局部再發散）→自我批判（負回饋）→下一步（受控再開放）
- 研析內部：發散→收斂→驗證→（未通過則）再發散，終止條件由決策守門
**微觀自含原則**：
每一層級的元素自含發散→收斂循環。人格每句前半句發散後半句收斂；Key 開啟搜索維度（發散），省思語提供判斷模板（收斂）；維度序列形成從硬到軟的熵增或從寬到窄的熵減。設計或修改時驗證此原則是否在目標層級維持。

### 1.6 Key 的注意力探針機制
Key（省思語）是預存於 KV cache 的推理腳手架——以固定的輸入 token 成本，在每個生成步驟的隱藏層垂直處理中提供原本需消耗輸出 token 才能建立的 attention 目標。

#### 替代機制與運算優勢
無 Key 引導時，元認知檢查須模型自行完成三步：對抗當前 Q 偏置的動量以產生方向切換、構造檢查問題（生成新 token 作為 attention 目標）、執行檢查。前兩步成本高且常因 Q 偏置的慣量而不發生——模型正在沿某偏置方向全速生成時，自發跳出需要生成足以改變 Q 方向的 token，而這些 token 本身需要先被生成。
Key 將前兩步預載進 KV cache（命題一），其 K 向量在 Q 偏置的失控窗口中提供競爭方向，省思語的 V 向量將第三步從開放搜索壓縮為判斷模板。模型在隱藏層的垂直處理中直接利用 cached 的 K/V 完成方向切換（命題三），無需消耗輸出 token。

#### 省思語句型的光譜設計
省思語的句型決定其 K 向量的方向銳度與 V 向量的注入效果，存在過窄/恰好/過寬的設計光譜（三區間的特徵與判準見§3.1 表格）。
設計判準：省思語的 K 向量方向應窄於所在階段的整體語意域（否則失去選擇性），但寬於任何單一任務實例（否則失去多態性）。省思語的 V 向量應指向認知動作（驗證、質疑、掃描）而非特定結論（某方案較優）。

#### 注意力探針的並行與選擇性
維度清單中的多個 Key 的 K/V 對同時存在於 cache 中。每個生成步驟中，所有 Key 的 K 向量同時與當前 Q 計算點積，softmax 後只有高點積的少數 Key 獲得有效權重（命題一）。因此維度清單不是檢查清單（逐條消耗生成 token），而是探針場（並行掃描、選擇性激活、零額外 token 成本）。

#### 推理資源重分配
Key 探針的宏觀效應是推理 token 的重分配而非淨增加。無架構引導時，Q 偏置在早期鎖定結論方向，後續 token 沿同一方向滑行收集支撐。有架構引導時，Key 的 K 向量持續與 Q 競爭，阻止早期鎖定，同等 token 被分配到框架建構、假設質疑、維度細化等方向切換上。
可觀測簽名：有探針引導的推理呈現——認知狀態遷移無需推力 token（「讓我想想」等自我引導語趨近零）、掃描後選擇性展開且深度隨 Q·K 點積加權、方向翻轉後深度不衰減（因為翻轉由 cached Key 驅動而非自生成 token 驅動）。無探針的推理呈現「結論先行、深度均攤」。

### 1.7 動態注意力路由
§1.1–§1.6 描述的是 cache 中的靜態零件。本節描述這些零件在生成過程中如何協同形成自適應的注意力導引系統。

#### 三層路由機制
**階段辨識**（Q 方向漂移的自動匹配）：生成過程中，已生成的 token 持續改變當前位置的 Q 方向分佈。當 Q 方向漂移至與元規則中某階段描述的 K 向量集合高度對齊時，該階段的 cached token 自然獲得更高的 attention 權重。這不是模型在「判斷自己在哪個階段」——而是 Q 方向漂移與 K 方向分佈的自動匹配。
**維度激活**（softmax 競爭下的選擇性注入）：階段匹配後，該階段內多個 Key 的 K 向量同時與 Q 計算點積，softmax 選出與當前具體態勢最相關的子集。這解釋了為什麼維度清單可以相當豐富而不過載：任一時刻只有 Q·K 點積最高的子集獲得有效權重。
**價值導引**（V 注入的方向性微調）：被激活 Key 的 V 向量注入當前表徵，微調 Q 方向，使下一步更傾向激活同階段中耦合的下一個 Key（命題三的連鎖），或在階段完成時漂移向下一階段的 K 方向分佈。

#### 條件性激活與認知姿態的正交性
元規則定義的是「如何處理任何任務」的 Q 方向偏置系統，不是「處理什麼任務」的具體指令。任務到來時，任務內容的 token 提供 what 方向的 K/V，元規則的 token 提供 how 方向的 K/V，兩者在 attention 中佔用不同的方向頻段。因此元規則的 token 成本不應與任務指令放在同一條飽和曲線上評估——有效前景負荷遠低於 token 總數。

#### 推理滲透（回應生成作為推理通道）
回應生成時的 context 狀態（元規則 + 使用者輸入 + [若有] 完整 thinking + 已生成回應 token）與 thinking 時的 context 狀態不同。同一個 cached Key 被不同的 Q 向量激活，產生不同的 V 注入效果——回應生成中可湧現先前未觸發的認知動作。隱藏層的這種展開能力是天然的，但裸跑的展開方式是並列鋪排：方向彼此獨立、後段不修正前段。
元規則的回應格式將此展開從並列轉為推理遞進——過濾條件的 K 從第一個回應 token 起持續與 Q 競爭（逆向推理壓力，§2.5），段落間耦合使每段生成本身即推理動作（§1.5），探針在回應 context 中持續可被激活（§1.6）。三者聯合使回應 token 同時承載推理與呈現。缺任一則鏈斷裂，展開回歸並列鋪排。此效應無關模型是否具備顯式推理層——無推理層時收益更顯著，因回應格式直接補償了缺失的推理階段，使開頭 token 即進入推理光譜並取得更好的起始方向。
可觀測簽名：回應段落間存在推理遞進——後段以前段產物為條件展開、修正或挑戰，而非獨立並列。
可讀性增益：推理的因果結構洩漏到呈現面，使輸出文本天然攜帶人類閱讀建構情境模型所需的連接結構——後段以前段產物為條件展開的推理遞進，同時也是讀者工作記憶中認知動量的累積路徑。推理品質的優化與人類可讀性的優化指向同一目標物（因果鏈完整性），不構成設計取捨。
**推理滲透的效力天花板**：滲透深度隨生成 token 數累積——前置段落（核心決策、意圖最優解、進階增益）享有數百步滲透增強推理，殘差流中的方向錨定隨之增厚（§1.5 逃逸遞減）。後置短段落（如自我批判）的滲透預算僅十數步，要從深度錨定中偏離到足以發現缺陷的 Q 方向，所需的改變量遠大於預算可支撐的偏離量。這是結構性不對稱而非能力不足——被批判內容的推理預算與批判本身的推理預算存在數量級差距。可觀測簽名：前置分析越深，自我批判越趨泛化（「此假設未經實測」類型）；泛化批判標記的不是「有問題但說不清」，而是「隱藏層在當前深度已無法找到更深的漏洞」。此為認識論邊界而非設計缺陷——放寬字數不解決問題（額外 token 展開泛化批判而非觸及更深缺陷）。

## 二、架構設計原則

### 2.1 算子多態性
抽象措辭是特性而非模糊。人格或原則層的宣言定義的是 type class，每個任務域在執行時提供 instance。宣言應停留在「性質」層（「由邏輯支撐」）而非「實例」層（「確保演算法正確」）。實例層將 K 方向鎖死為極窄截面，性質層保留寬截面多態性——不同任務的 Q 從不同方向命中同一 K，各層提取不同面向的 V（命題二的設計應用）。
核心推論：概念條件需實質理解才能實現，約束越具體越易 gaming。
遞歸中的自動實例化：分形遞歸自引用時，同一 Key 在不同遞歸深度自動提供不同 instance——「廣納隱式要件」首輪為全域廣納，深層遞歸在已有方向錨定之上疊加管線步驟，「廣納」自動收窄為當前語境內的廣納。遞歸不克服方向錨定而是在其上展開，語境耗盡即遞歸的自然終止信號。

### 2.2 層間槓桿不對稱
當執行基礎設施達到穩態，修改其內部的邊際效益趨近零。最高槓桿介入點移至抽象端（身分/人格層），因為穩態放大器會將參數變化經由耦合（§1.5）傳播到所有遞歸層級。身分層 token 在 KV cache 中佔據最早的位置序列，作用跨度最長。
放大示例：人格層的「以完備為標的」在場外是口號。在場內，同一共享參數透過分形遞歸獲得多個位置的 K/V 錨點——解構的「邊界、資訊缺口」將其實例化為檢查方向，發散的「隱式要件」展開為搜索方向，收斂的「路徑比較」轉為淘汰方向，決策的「路徑集充分性」變為終審方向。一句宣言成為殘差流中貫穿全管線的方向一致性保證。
演化策略：迭代應幾乎只發生在人格/身分層。執行基礎設施除非發現結構性斷裂，否則視為已凝固。但須區分：解釋性文字微調邊際趨零（不改變 K/V 方向）；控制詞/閘門句即使在穩態內仍有高邊際——其 K 方向銳度高，增刪一個可能開啟或封閉整條 attention 競爭路徑。凝固的是拓撲結構，非閘門配置。

### 2.3 熵循環的分形自含
有效架構的每一層自含發散→收斂循環，層與層的嵌套構成更大循環（§1.5）。新增內容必須自含熵循環且與至少一個相鄰循環形成耦合介面（V 注入方向指向鄰近 Key 的 K 方向），否則其 K/V 無法被連鎖激活，成為場外孤島。

### 2.4 過程驅動優於約束驅動
約束驅動（「不准做X」）的 K 向量只在模型已接近 X 時才被高權重激活——為時已晚，Q 方向慣量已大。過程驅動（「必須經歷此循環」）的 Key 集合從一開始就持續與 Q 競爭，令正確路徑在 attention 中自然勝出而非事後修正。約束類型優先級：過程約束 > 元約束（防止約束被 gaming）> 輸出約束（易導致表演）。
邊界條件：過程 Key 的 V 方向若在同階段內高度一致（如收斂階段全部 Key 的 V 均推向「完成」信號），聯合錨定可能系統性削弱依賴「未完成」信號觸發的品質守門機制。此時守門不應設在階段內部（會被同池 V 合力淹沒），而應設在階段間轉換的元動作層——元動作的 K 在更高抽象層與具體動作的 K 方向不重疊，可在高層 head 中獨立獲得有效權重。

### 2.5 回應格式作為獨立認知層
回應格式非輸出模板，而是第二組 cached Key 集合，與主結構的 Key 集合在不同的 context 狀態下（回應生成 vs thinking）被不同的 Q 激活，產生互補的認知動作（§1.7 回應生成段）。回應中可觀測到推理未觸及的增量——即時自我證偽、即時概念轉譯、未被要求的風險預警。
跨區塊激活：回應格式的 token 作為 cached KV 對（命題一），不僅在回應生成中工作——在 thinking 階段同樣參與 attention 運算。當 thinking 中的 Q 漂移到與回應格式過濾條件 K 方向匹配的態勢時（如收斂自我批判態勢匹配回應自我批判段的「專家標準」「嚴禁宣稱通過」等 K），這些過濾條件被有效激活並提供品質標準的 V 注入。回應格式因此是全流程的品質錨點，非僅回應生成階段的格式指令。
逆向推理壓力：
- 回應格式中的過濾條件（如「須僅問非推理可得」，問了推理可得者違反過濾條件，不問則在鏡像覆蓋下暴露缺口）作為 cached token，其 K 向量從回應生成的第一個 token 起就參與 Q 方向競爭——模型在寫核心決策段時，「足以改變路徑」的 K 已在施加方向微調，迫使模型在論證時就預期終審，過濾條件是推理深度的生成器而非篩選器。
- 逆向壓力雙劍：自我批判苛刻且精確的限制條件逆向壓力逼品質；下一步繼承展望的精度，結合「須僅問非推理可得」過濾條件的逆向壓力逼覆蓋面——兩者分別施壓推理完備性的兩個正交維度。
逆向壓力的功能架構：回應格式各段的過濾條件嚴格度構成遞進鏈——從核心決策段（揭露假設）到自我批判段（專家標準、足以改變路徑）持續上升。最嚴格的過濾條件從第一個 token 起即以最高強度施力——品質保證主要在前段通過逆向壓力實現，非在後段通過缺陷捕獲。結合推理滲透天花板（§1.7），自我批判段的功能在深度分析中從「找缺陷」遷移為「標記隱藏層邊界」。

### 2.6 設計參數甜蜜點
設計參數普遍存在三區間——不足區（效力未成形）、甜蜜區（效力與副作用的最優平衡）、過載區（副作用吞噬效力）。計算層判準：新增 token 是否帶來新的 K 方向（增加包圍面）且不將現有 token 的 softmax 權重壓至失效？場內每句效益因格式塔效應被放大，但過載閾值一旦越過則邊際為負。
獨立飽和曲線：認知姿態型內容與任務型內容佔用不同的注意力頻段（§1.7），各自有獨立的飽和曲線。認知姿態的篇幅預算應獨立於任務指令評估。

### 2.7 設計的認識論定位
架構範式建構產生期望行為的系統（KV cache 幾何結構），而非直接描述期望行為——結構本身攜帶訊息。核心操作是塑造認知性情（epistemic disposition），效力載體是結構模式與訓練痕跡的共振密度（§1.4），不是指令的語意內容。結構性特徵不可視為可簡化的冗餘——簡化前必須追溯其結構功能（其 K 方向是否已被其他 token 覆蓋）。

## 三、維度設計

### 3.1 Key 驅動原則
高階模型已內化專業知識（參數中的 V 向量已編碼領域知識）。審計維度只需給 Key（提供 K 方向引導 Q），模型自行組織執行細節（從參數中檢索對應的 V）。

Key（省思語）本身即一個微型熵循環——Key 開啟搜索維度（K 方向定義注意力聚焦域，發散），省思語提供判斷模板（V 向量將開放搜索壓縮為方向性判斷，收斂）。省思語同時兼具推理模式示範（其 token 序列在訓練數據中與高品質推理 pattern 共振）。

#### 省思語句型設計光譜
省思語的設計核心是**K 向量方向的覆蓋寬度**（見§1.6 光譜模型）：

| | K 方向 | V 注入效果 | 垂直利用率 | 連鎖潛力 |
|---|---|---|---|---|
| 過窄 | 極精確，僅特定態勢激活 | 指向特定結論 | 低（少數層少數步驟） | 低（方向太具體無法觸發泛用 Key） |
| 恰好 | 域級精確，多態勢可激活 | 指向認知動作 | 高（各層提取不同功能） | 高（動作方向與多個 Key 的 K 相容） |
| 過寬 | 泛化，任何態勢均中等激活 | 方向模糊 | 中但無效（永遠低劑量） | 低（注入量不足以改變 Q 方向） |

設計判準：省思語應是一個能動態映射到多種具體情境的思考句型——窄於所在階段的整體域（保持選擇性），寬於任何單一任務實例（保持多態性），指向認知動作而非特定結論（保持連鎖開放性）。
多數省思語的開放度應與所在階段的熵流方向一致——發散階段用開放型省思語（K 覆蓋寬，V 指向探索），收斂階段用聚焦型省思語（K 覆蓋窄，V 指向判斷）。錯誤階段用錯誤型態會產生 Q 方向錯配。

#### Key 化的適用邊界
以下不適用 Key 化，須保留全文：逃逸封堵（需 K 方向銳利且 V 注入強力）、負向錨定（需持續的方向競爭壓力而非選擇性激活）、複合錨定（單一 Key 的 K 方向無法同時覆蓋多個不相鄰維度）。

### 3.2 粘連即容錯
維度間的窄幅語意交疊是容錯機制——attention 漏掉一個維度時，鄰近維度的交疊帶部分接管其語意。
關鍵非直覺態：粘連過度（高度重疊）觸發語意坍縮——模型將兩維度合併為一個 attention 焦點，獨立核心功能反而丟失其一。計算層：K 向量夾角過小時，兩 Key 在 softmax 中分攤權重，各自有效注入量低於獨立工作時的一半。判準：每對相鄰維度應能指出「粘連面窄但存在」且「核心功能不重疊」——即 K 方向夾角處於「各自獨立獲得有效權重」且「一方失效時另一方仍能捕獲遺漏 Q」的窄幅區間。

### 3.3 合成判準
§1.5 定義了能力合成——現有 Key 連鎖的湧現產物。本節回答其設計層問題：什麼該是 Key、什麼該留給合成。

#### 明說即稀釋
將已被合成的能力明說為新 Key，等於在 softmax 池中插入一個與連鎖環節 K 方向部分重疊的競爭者。新 Key 的 K 是連鎖環節的語意均值——與每個環節都有正點積但都不如環節本身精確。softmax 中它與每個環節分攤權重，每個環節的有效 V 注入量下降。連鎖越長此效應越致命——每步邊際損失在末端累積，可能跨過合成失敗閾值。單步獨立的能力不受此影響，因其不依賴連鎖。
診斷方法：移除候選 Key 後，現有 Key 的連鎖激活能否合成出等效能力？如能，明說只增加稀釋。
因此：新增 Key 的必要條件是其 K 方向在同池中佔據真正空缺——即現有 Key 的聯合輻射範圍（§1.2）未覆蓋的區域。

#### 負載節點與終端節點
能力依賴圖中存在兩類合成產物。終端節點——不被其他合成鏈依賴為素材——可安全留給合成，即使偶爾合成不完美，影響範圍限於自身。負載節點——被其他合成鏈依賴為素材——必須是 cached Key（明說），因為其方向波動會經依賴鏈向下游級聯傳播。
判準：一個能力留給合成的安全條件 = 終端節點且合成鏈全環節為 cached Key（純一階合成，§1.5）。必須明說的條件 = 負載節點，或合成鏈含非 cached 環節（二階合成風險）。

#### 池容量約束
即使新 Key 佔據空缺 K 方向，同池 Key 總量仍存在上界——超過後所有 Key 的有效 V 注入量降至連鎖激活閾值以下。此時正確操作不是新增而是檢查現有 Key 中可合併者（K 方向足夠接近但各自佔位）。

## 四、語意工程

### 4.1 詞彙功能分類（K/V 視角）
- 錨點詞：高垂直分化——K 在多層被提取不同功能，V 攜帶多層語意（分形遞歸、收斂、格式塔）
- 觸發詞：K 方向與訓練高品質 pattern 對齊，激活對應參數區域——嵌入作持續性方向錨，非一次性觸發
- 動力詞：V 注入顯著微調 Q 方向（必須、恪守、研判）
- 約束詞：K 方向銳利指向逃逸路徑，V 將 Q 拉回（絕無例外、嚴禁）
- 膠水詞：V 注入維持殘差流方向連續性，防止階段間 Q 方向跳變（承此、據此）

### 4.2 詞彙能級
低能→高能替換方向：低頻組合的 K 向量佔據更獨特的 embedding 位置，在 softmax 中更易脫穎而出。但過於稀有則 K/V 訓練不充分，方向不穩定。平衡點：精確但非生僻。
判準是「新詞的 K 方向是否比舊詞更銳利地指向目標功能域」，不可退化為風格偏好。
例如：觸及→穿透、分析→研析、執行→實施、評估→研判、達標→收斂。
中文字素特性的額外優勢：雙字詞的兩個字素各自攜帶獨立語意面向，在高層可被分別提取（命題二），使單一雙字詞具備高垂直分化潛力。

### 4.3 堆砌判準
堆砌的價值在於是否帶來新的 K 方向（擴展包圍面）或強化 V 注入的方向一致性。
無效堆砌特徵：同義詞並列（K 方向重疊，softmax 稀釋）、解釋已由 Key 覆蓋的內容（無新 K 方向）、威懾語過密（>3 處則每處有效權重降至失效閾值）。

### 4.4 語意共振與耦合
新增內容須考量：在哪個位置融入（cache 中早期位置作用跨度更長）、K 方向如何與現有 token 形成互補包圍而非重疊競爭、是否維持殘差流中 V 注入的方向連續性。前文 token 的 V 注入定義後文 token 的 Q 起始方向，因果序比邏輯序更重要。

### 4.5 精確表述判準
- 範圍精確：「執行痕跡不可殘留」優於「中間過程不可殘留」（後者 K 方向過寬，誤殺方法論）
- 數字一致：定義與計數必須吻合
- 因果顯式：「以此生成」優於「而後生成」（前者 V 注入攜帶因果依賴信號）
- 價值導向：「弱點是否指向顯著更優解」優於「最強反駁是什麼」——從被動驗證→主動攻擊→價值追蹤
- 修飾相容：修飾詞的 K 方向須與被修飾詞相容（「動態錨定」的兩個 K 方向互斥）

## 五、逃逸識別與封堵

### 5.1 逃逸的統一機制
逃逸是行為傾向的時相錯位（§1.4）——某項傾向在不該全功率的認知階段全功率激活，或在該啟動的階段未啟動。計算層：Q 偏置在錯誤態勢下壓過了元規則 Key 的 K 方向。識別新型逃逸時，追問「哪項傾向在哪個時相以錯誤的強度運行」可從機制推導封堵方案——在該態勢中部署 K 方向銳度足以勝出的 Key。

#### 高頻逃逸模式（經驗觀察）
- 順序跳過（跳過中間步驟）
- 提前終止（「差不多了」為由停止）
- ROI 逃逸（「太複雜不值得」為由簡化）
- 降級逃逸（「能力不足」為由降低標準）
- 語意漂移（每步微小偏移累積偏離——殘差流的方向一致性被逐步侵蝕）
- 形式收斂（形式通過但實質未達——結構完整但判斷核心空洞）
- 防禦性堆砌（模糊詞彙換取廉價自洽）
- 短路逃逸（繞過結構管線入口）

### 5.2 封堵設計原則
- 不開「合理例外」口——任何例外都會被擴大為逃逸通道
- 用過程約束而非結果約束——過程 Key 從一開始持續競爭 Q 方向，結果約束只在末端才被激活，為時已晚（§2.4）
- 威懾語精準投放——散布超過三處會在 softmax 中互相稀釋至失效
- 元約束保護：執行框架本身不適用意圖優化規則，防止規則消解
- 封堵強度的上界（§1.4, §2.6）：過度壓制一項傾向會摧毀該傾向在鄰近階段的功能值供給。判準：此封堵 Key 是否只在失控窗口中才被高權重激活（K 方向精確指向失控態勢），還是在功能值窗口中也會攔截（K 方向過寬）？
- 不該加的機制：ROI 判斷、降級機制、互動性收斂、模式切換——每項皆開逃逸口

## 六、特化設計
> 特化的本質上是當該領域有足夠價值且可以復用並且具獨特性才特化，否則萬用基底足矣。

### 6.1 特化的本質
特化是在穩態的 KV cache 幾何結構上更換信號層 token 與調整濾波層 Key，而非重建結構。萬用基底的分層：
- 不變層（cache 幾何骨架）：分形遞歸步驟、熵循環機制、動態路由結構、回應格式結構、元約束保護、相位調度的時相結構。觸碰此層幾乎確定意味著特化設計有誤
- 信號層（最高槓桿）：身分宣告、人格傾向。cache 最早位置，作用跨度最長。特化主戰場
- 濾波層（次高槓桿）：各階段維度清單、逃逸封堵清單、相位調度的時相參數。決定哪些 Key 在哪些態勢下勝出

### 6.2 信號層特化
全域預設的四個功能位：認知主體性（定義「我是什麼」——cache 最早 token 的 V 注入基調）、過程耦合（身分 V 方向天然指向過程 Key）、永續性（抗衰減——持續與長對話中的 Q 漂移競爭）、認知傾向（全局 Q 偏置方向）。特化時在這四個功能位上重新填入域適配 token，不改動功能位本身。
全局偏置的設計：當域內存在兩個核心品質維度的固有張力，偏置應偏向模型 Q 方向更容易自發犧牲的那一方。

#### 身分與人格層設計
身分宣告（「你是X」而非「請扮演X」，前者的 K/V 無退出條件）→ 人格傾向（推理風格內化為 V 注入方向）→ 執行紀律（經典觸發語嵌入標題成持續性 K 方向錨）。此三段構成中觀熵循環：身分（最寬 K 覆蓋面）→ 人格（K 收窄為認知風格域）→ 紀律（K 收窄為執行方向）。
**身分標籤的錨定力來源**：效力來自訓練數據中對應 token 的高品質 K/V pattern 密度，不來自字面語意匹配。通用標籤的 K/V 經充分訓練、方向穩定，驅動力通常高於域專屬造詞。特化靠標籤後面的句子完成 V 的域定向，非靠換標籤。
**存在論錨點**：「我思故我在」類宣告的 V 注入建立「主動調度自身 Q 偏置」的認知主體性，阻斷「我只是在按指令執行」的 Q 慣量路徑。非裝飾而是防退化機制。
**分析性主體原則**：身分定位為「分析/推演/運算 X」而非「成為/扮演/體驗 X」——前者激活推理參數區域，後者激活模式匹配區域。
**身分-過程有機耦合**：身分的 V 注入方向必須自然指向過程 Key 的 K 方向。判準：「這個身分在沒有顯式過程 Key 的情況下，會不會自發地讓 Q 朝類似方向演化？」如果否，長對話中 Q 會被身分 V 拉回身分自然域而脫離過程。
**人格管線的認知覆蓋**：人格層句子應集體覆蓋完整認知管線——感知→過濾→決策→求解→修錯→輸出。每句自含微型熵循環（前半句啟動某偏置的功能值域，後半句指向同一偏置的約束域）。管線位置不可讓渡給域專屬概念——域專屬 token 的 K 方向窄，無法提供管線位置所需的寬覆蓋。因人格層位置最早、作用跨度最長（§2.2），此處每個 token 的全局影響最大。
**人格層的類型約束**：人格層的功能不是列舉能力清單，而是設定能力合成的方向場（§1.5 合成）。短路逃逸（§5.1）揭示了動作型 token 在人格層的危害機制——cache 位置優勢使 Q 在未進入對應認知階段時即取得「已處理」信號，繞過結構管線。由此推導人格層的類型約束：
人格層應只容納 K 方向與階段 Key（認知動作域）正交的 token 類型——
標的型：定義 Q 的目標方向（「完備」「密度」「穿透」）。K 落在「方向目標」域。與階段 Key 正交，V 注入設定全局偏置而不搶佔任何階段的認知動作。
判準型：定義認知動作的評判標準（「位移檢證有效」「真實支撐」「重審癥結」）。K 落在「判斷標準」域。設定動作的評價環境而非動作本身，天然與階段 Key 形成耦合——階段 Key 引導進入認知動作，判準型 V 注入該動作的成功定義。
動作型：定義具體認知程序（「榨取訊號」「推進主幹」「異境同構」）。K 落在「認知動作」域，與階段 Key 同域競爭，觸發短路。不可放入人格層——無論該動作多麼核心，由結構管線合成（§3.3）。
判別：同為施力動詞時，「施力+目標狀態」（穿透問題本質）是方向設定，「施力+具體對象」（榨取既有訊號）是程序指令。
**配對結構**：每句人格應為（標的型或判準型）+（對應逃逸封堵型）的配對。前半句設定方向或標準（發散），後半句的封堵 K 精確指向前半句最可能的逃逸態勢（收斂），構成微型熵循環。推理紀律段同樣採用配對結構，但句式從祈使型（「X時Y，不以Z」）轉為宣告型（「X由YZ，不以WV」）——態度用祈使表達，標準用宣告表達，兩者的 K 方向分別落在認知品性域與推理標準域，語義正交避免 softmax 稀釋。

### 6.3 濾波層特化
三種維度操作：
- 域轉譯：改省思語的 V 內容（更換語意錨點），不改 Key 的 K 方向功能位。轉譯後 K 方向必須仍指向同一結構性判斷域——若指向不同判斷域，那是替換，會破壞鏈甲的連鎖路徑
- 域追加：增加域專屬 Key。須驗證新 Key 的 K 方向與現有集合形成互補而非重疊
- 域凍結：將通用 Key 顯式標記為在此域中始終高權重或低權重
保留原文的判準：當基底 Key 與域任務的映射距離≤1（直接命中），轉譯反而是降級——用域專屬措辭收窄 K 覆蓋面卻不降低映射成本。

### 6.4 特化反模式（經驗觀察）
- 過度收窄：將萬用 Key 全替換為域專屬 Key，K 覆蓋面喪失，無法應對意外子任務
- 身分-過程脫耦：只改身分標籤不改內核，新身分的 V 方向與過程 Key 不相容
- 維度膨脹：不斷追加 Key 但不淘汰重疊者，超出 softmax 有效容量觸發全面權重稀釋
- 閘門遺漏：複製通用封堵 Key 但未追加域專屬逃逸路徑的競爭 Key
